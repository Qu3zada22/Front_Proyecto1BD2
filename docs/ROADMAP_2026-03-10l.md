### BUG-01 — `ordenes.updateStatus()` eliminar race condition con filtro atómico

**Archivo:** `apps/api/src/ordenes/ordenes.service.ts`
**Prioridad:** MEDIA

**Cambio:**

En `updateStatus()`, reemplazar el `findByIdAndUpdate` final con `findOneAndUpdate` que incluya el estado actual como condición del filtro. Esto garantiza atomicidad: si otro request cambió el estado entre el read y el write, el update retorna `null`.

```typescript
// ANTES (línea ~142):
const updated = await this.ordenModel
  .findByIdAndUpdate(id, update, { new: true })
  .exec();
if (!updated) throw new NotFoundException("Orden no encontrada");

// DESPUÉS:
const updated = await this.ordenModel
  .findOneAndUpdate({ _id: id, estado: (orden as any).estado }, update, {
    new: true,
  })
  .exec();
if (!updated) {
  throw new BadRequestException(
    `Conflicto: el estado de la orden cambió antes de aplicar la transición. Reintente.`,
  );
}
```

Nota: el primer `findById` sigue siendo necesario para validar la transición con el mapa `TRANSICIONES` y lanzar error descriptivo si es inválida.

**Tests a actualizar:** `apps/api/src/ordenes/ordenes.service.spec.ts`

- Test existente de `updateStatus` verifica `findByIdAndUpdate` → cambiar a `findOneAndUpdate` con matcher de filtro `{ _id: id, estado: currentEstado }`
- Añadir test: "should throw BadRequestException when estado changed between read and update" (mock `findOneAndUpdate` returning null después de `findById` exitoso)

### BUG-02 — `resenas.remove()` envolver en transacción ACID

**Archivo:** `apps/api/src/resenas/resenas.service.ts`
**Prioridad:** BAJA

**Cambio:**

Envolver las 3 operaciones de `remove()` en sesión/transacción, usando el mismo patrón que `create()`:

```typescript
async remove(id: string): Promise<{ deleted: boolean }> {
    const session = await this.connection.startSession();
    session.startTransaction();
    try {
        const resena = await this.resenaModel.findByIdAndDelete(id).session(session).exec();
        if (!resena) throw new NotFoundException('Reseña no encontrada');

        if (resena.restaurante_id) {
            const [stats] = await this.resenaModel.aggregate([
                { $match: { restaurante_id: resena.restaurante_id, activa: true } },
                { $group: { _id: null, avg: { $avg: '$calificacion' }, count: { $sum: 1 } } },
            ]).session(session);
            await this.restauranteModel.findByIdAndUpdate(resena.restaurante_id, {
                $set: {
                    calificacion_prom: stats ? Math.round(stats.avg * 10) / 10 : 0,
                    total_resenas: stats?.count ?? 0,
                },
            }, { session });
        }

        if (resena.orden_id) {
            await this.ordenModel.findByIdAndUpdate(resena.orden_id, {
                $set: { tiene_resena: false },
            }, { session });
        }

        await session.commitTransaction();
        return { deleted: true };
    } catch (err) {
        await session.abortTransaction();
        throw err;
    } finally {
        await session.endSession();
    }
}
```

- Verificar `commitTransaction` en happy path, `abortTransaction` en error

2. **usuarios.service.ts** — inyectar los 3 modelos e implementar verificación:

```typescript
constructor(
    @InjectModel(Usuario.name) private usuarioModel: Model<UsuarioDocument>,
    @InjectModel(Orden.name) private ordenModel: Model<any>,
    @InjectModel(Resena.name) private resenaModel: Model<any>,
    @InjectModel(Restaurante.name) private restauranteModel: Model<any>,
) { }

async remove(id: string): Promise<{ deleted: boolean }> {
    const [ordenesCount, resenasCount, restaurantesCount] = await Promise.all([
        this.ordenModel.countDocuments({ usuario_id: id }),
        this.resenaModel.countDocuments({ usuario_id: id }),
        this.restauranteModel.countDocuments({ propietario_id: id }),
    ]);
    if (ordenesCount > 0 || resenasCount > 0 || restaurantesCount > 0) {
        throw new BadRequestException(
            'No se puede eliminar: el usuario tiene datos asociados (órdenes, reseñas o restaurantes).',
        );
    }
    const result = await this.usuarioModel.findByIdAndDelete(id).exec();
    if (!result) throw new NotFoundException('Usuario no encontrado');
    return { deleted: true };
}
```
