## Prioridad Crítica (debe hacerse ANTES del demo)

### 1. BUG-01: Decimal128 en `ordenes.service.create` → NaN en aritmética

**Archivos:** `ordenes.service.ts`, `ordenes.service.spec.ts`

**Causa:** `03_menu_items.js` almacena precios como `Decimal128`. `.lean()` retorna Decimal128 sin type-casting. `Decimal128 * number = NaN`.

**Síntoma en producción:** Después de `POST /api/seed`, crear una orden via `POST /api/orders` produce `total: null` y `subtotal: null` en cada ítem.

- [ ] En `ordenes.service.ts`, líneas 63-66: cambiar acceso directo a `dbItem.precio` por `parseFloat((dbItem as any).precio?.toString() ?? '0')`:

```typescript
const precio = parseFloat((dbItem as any).precio?.toString() ?? '0');
return {
    ...
    precio_unitario: precio,
    precio: precio,
    subtotal: precio * i.cantidad,
    ...
};
```

- [ ] En `ordenes.service.spec.ts`: agregar test que simula Decimal128 como precio y verifica total correcto:

```typescript
it("should handle Decimal128 precio from DB (BUG-01 Decimal128)", async () => {
  const decimal128Price = { toString: () => "50", _bsontype: "Decimal128" };
  mockMenuItemModel.find.mockReturnValue({
    lean: jest
      .fn()
      .mockResolvedValue([
        {
          _id: new Types.ObjectId("507f1f77bcf86cd799439031"),
          nombre: "Burger",
          precio: decimal128Price,
        },
      ]),
  });
  // ... setup dto con 1 item, cantidad 2
  // expect total = 50 * 2 = 100, NOT NaN
});
```

**Estimación:** ~5 líneas de código + ~15 líneas de tests
